Proposal

Source Control Explorer defaulted to the right side so that when switching branches or checking out previous commits, you can easily access the changes.

Make use of Source Control Explorer Pane:

  • Branch Name displays up top
    • Defaults to what branch you are currently in
    • Click opens either a dropdown to switch to another or opens Branch UI
    • Should we allow you to not have to jumps to branch and just preview contents below in order to not have to reload project when just trying to investigate?
      • Selecting new branch name just reflects the data in the pane
        • Would need a Jump To option/button saying viewing/current
    • Checkout - Works with left clicking node
      • Brings you to state of project at time of commit
      • Project Files update/revert
    • Merge - Works with left clicking node
      • Opens Merge Dialog
  • Pending Changes - Keep as is for now
  • Graph View (DEFAULT VIEW (New Home)) - Reflect a graph of the branches
    • Graph - Should be interactive to quickly jump to different states of the project
      • Lines reflect branches
      • Nodes on lines show each commit
        • Hover/Click displays the Commit ID (whole or last 8?) with option(s):
          • Checkout - Brings to state of commit; Project files update/revert
          • Merge - Brings up Merge Dialog
    • Actions column - displays in Graph View
    • Message - Commit comment
    • Date - of commit
  • File View - Bottom half of pane which is dynamic with left click selection of node up top in graph view
  • Repo View - Reflect a snapshot of files in the repo (Optional, may be overkill)?
    • If so, split pane
    • Hover - Date and commit ID
    • Double click - nothing overkill right now
  • Commit Log View- An alternative view, showing text list of commits with Commit ID (Non-interactive?)
  • New Diff UI
    • Non-Modal
    • Split screen
    • Simple color-coding
    • Simple UI Specific to SCM, This case is git
